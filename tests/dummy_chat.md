# Rust 借用检查器探讨

> **平台**: Obsidian
> **ID**: 123456789

### 🧑 User

能简单说明一下 Rust 中借用检查器（Borrow Checker）的核心原则吗？

### 🤖 Assistant

当然可以！Rust 的借用检查器主要基于以下两个核心原则（在任何给定时间只能满足其一）：
1. **多个不可变引用** (`&T`)：你可以有任意数量的指向同一数据的不可变借用。
2. **唯一的并且是可变的引用** (`&mut T`)：只能有一个指向该数据的可变借用。

这从根本上杜绝了数据竞争（Data Races）。

### 🧑 User

所以如果我有一个可变借用，就不能再创建不可变借用了是吗？

### 🤖 Assistant

完全正确。如果你已经创建了一个可变借用，在它的生命周期结束之前，编译器会阻止你创建任何新的借用（无论是可变的还是不可变的）。
