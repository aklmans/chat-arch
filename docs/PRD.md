# 聊天资产管理 CLI 工具 (ChatArch) - 产品需求文档 (PRD)

> **定位**：面向个人/团队的 AI 聊天资产归档、检索、分析、复用与安全管理系统，以 CLI (命令行) 为核心交互形态。

---

## 1. 产品目标

它不只是“存聊天记录”，而是帮助用户完成：
- **统一归档**：来自 ChatGPT、Claude、Gemini、Cursor、Copilot、终端 Agent、微信群机器人等对话统一管理。
- **高效检索**：以后能快速找到某次有价值的对话。
- **资产沉淀**：把聊天变成可复用知识资产。
- **版本留痕**：记录提示词、上下文、输出结果的演变过程。
- **安全可控**：敏感聊天内容可加密、可脱敏、可本地管理。

## 3. 数据模型设计

为了支持长期管理，系统严格区分**会话级 (Session)** 与 **消息级 (Message)** 数据模型。

### 3.1 会话级字段（Conversation / Session）
- **ID**: 系统唯一标识 (UUID)
- **标题 / 主题**: 会话的主题名称
- **摘要**: 自动生成或手动编写的会话摘要
- **模型平台**: ChatGPT / Claude / Gemini / Cursor / Copilot / Ollama / Open WebUI 等
- **模型名称**: 具体使用的模型 (如 gpt-4o, claude-3.5-sonnet)
- **来源类型**: Web / CLI / API / IDE / Bot / 手动导入
- **创建时间 / 更新时间**: 时间戳
- **会话标签**: 编程、写作、产品设计、调研等 (支持多标签)
- **项目归属**: 关联的项目、仓库或任务
- **语言**: 中文 / 英文 / 混合等
- **状态**: 进行中 / 已归档 / 已收藏 / 已删除 / 已锁定
- **星标**: 是否为收藏的高价值对话
- **优先级**: 高 / 中 / 低
- **来源链接**: 原平台 URL 或原会话链接
- **外部引用**: 关联文档、Git 仓库、Issue、笔记等链接
- **成本信息**: Token 消耗数、估算费用、调用次数
- **隐私等级**: 公开 / 私密 / 敏感 / 高敏感
- **版本号**: 记录会话版本的演进
- **备注**: 用户自定义说明

### 3.2 消息级字段（Message）
- **ID**: 消息的唯一标识
- **会话 ID**: 关联的 Conversation ID
- **顺序号**: 在当前会话中的排序索引
- **角色**: user / assistant / system / tool / developer / function
- **发送者名称**: 用户、助手、工具、系统的具体标识
- **内容**: 消息的正文文本
- **内容格式**: text / markdown / code / json / html / image-desc
- **时间戳**: 发送/接收的具体时间
- **附件**: 关联的图片、音频、PDF、代码片段、文件本地路径
- **消息标签**: 关键问题、最终答案、可复用提示词、错误案例等
- **是否关键节点**: 标记出“重要回答”或“最终版本”
- **引用消息**: 回复目标的消息 ID
- **Token 信息**: 本条消息消耗的 Token 数量
- **执行结果**: 工具调用后的返回结果摘要
- **错误状态**: 成功 / 失败 / 中断 / 截断
- **哈希值**: 基于内容的 Hash，用于去重与完整性校验

## 4. 核心功能规划

### A. 分类与组织
- **标签系统**：支持自定义标签、多标签组合查询、基于规则的自动标签推荐。
- **项目空间**：支持按项目、客户、平台、用途对会话进行归档划分。
- **智能归档**：基于标题、模型、标签等维度设定自动归档路由规则。
- **精选集**：聚合高价值会话，如“最佳提示词案例库”。

### B. 导入能力 (核心竞争力)
- **手工录入**：单条命令录入、交互式问卷录入、剪贴板读取。
- **文件直导**：JSON, Markdown, TXT, HTML, CSV, PDF 文本提取。
- **平台适配器**：原生支持解析 ChatGPT, Claude, Gemini, Cursor 导出的归档包；支持 OpenAI API 日志等。
- **批量入库**：扫描指定目录及其子目录，自动推断格式并批量建立索引。

### C. 搜索与检索
- **基础搜索**：标题、全文、标签、平台、模型、日期精准组合筛选。
- **高级过滤**：仅搜索 user 内容、仅搜索代码块、仅搜索包含 URL/API 的内容。
- **布尔/模糊**：支持 AND / OR / NOT 逻辑表达式，容忍错别字的模糊匹配。
- **语义检索**：基于本地 Embedding 模型的向量搜索，支持按“意思”找上下文。
- **相似推荐**：查看记录时侧边/底部展示相似历史对话。

### D. 对比与版本化编辑
- **对比功能**：两组会话对比、不同模型对同一 Prompt 的回答对比、单条消息的版本 diff。
- **内容编辑**：提供会话元数据（标题、状态）及消息正文的修改能力，支持合并与拆分消息。
- **批量操作**：批量打标签、归档、修改隐私等级。
- **版本控制**：任何编辑行为将产生新版本，支持回滚及历史追踪。

### E. 导出与分发
- **基础导出**：HTML, PDF, Markdown, TXT。
- **结构导出**：JSON, CSV 供二次开发或统计使用。
- **打包与模板**：一键打包项目 ZIP；支持使用模板渲染（如阅读版、报告版、脱敏审计版）。
- **静态网站**：将会话集合导出为基于 VitePress/MkDocs 的静态知识库。

### F. 安全与治理体系
- **加密存储**：数据库级全局加密、会话级加密、单一敏感字段脱敏加密。
- **访问控制**：主密钥解锁机制，长时间无操作自动锁定控制台。
- **自动脱敏**：正则/规则引擎自动识别并遮蔽手机号、邮箱、API Key、密码等隐私数据。
- **审计与留痕**：完善的查看、导出、删除操作日志。
- **数据恢复**：支持回收站（软删除）、多版本加密增量备份。

### G. 高级 AI 资产化能力 (V2/V3 目标)
- **智能提取**：自动生成会话摘要、待办事项 (TODO)、风险点与决议。
- **提示词 (Prompt) 沉淀**：自动抽取高质量 User Prompt，沉淀为带参数的提示词模板。
- **关键帧标记**：自动或手动标记对话中的“最佳答案”与“最终结论”。
- **统计面板**：终端绘制会话走势、高频主题、Token 消耗排行图表。
- **知识图谱关联**：建立会话、文件、代码仓库之间的拓扑链接引用。

## 5. 命令行 (CLI) 交互设计

### 5.1 命令分组
```bash
chatasset add       # 快速手工添加单条记录
chatasset import    # 从文件或平台导出包导入数据
chatasset list      # 列表概览
chatasset show      # 查看具体会话详情
chatasset search    # 强大的多条件检索
chatasset edit      # 调用系统编辑器修改信息
chatasset delete    # 删除或移入回收站
chatasset export    # 按格式或模板导出
chatasset lock      # 锁定敏感会话或整个库
chatasset unlock    # 解锁库
chatasset stats     # 打印统计报表
chatasset backup    # 触发备份
chatasset restore   # 数据恢复
```

### 5.2 交互模式设计
- **交互式 (Interactive)**: 直接输入 `chatasset import` 会启动向导，逐步询问来源、标签等。
- **脚本化 (Non-interactive)**: 支持管道、静默与参数齐全运行。
  ```bash
  cat ./chat.md | chatasset import --format markdown --title "CLI设计讨论" --tags "cli,design" --yes
  ```

### 5.3 终端输出渲染
- **格式切换**: 使用 `--output table/json/yaml/text` 切换输出格式。
- **性能体验**: 分页显示长列表、大数据量懒加载（如 `show --messages 20`）。
- **Dry-run 模式**: 执行批量操作时支持 `--dry-run` 预览变更，配合 `--force` 跳过确认。

### 5.4 配置文件
支持通过 `~/.chatasset/config.yaml` 统一定义：默认存储路径、加密策略、外部编辑器别名、颜色主题等。
