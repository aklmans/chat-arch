# ChatArch 聊天资产管理 CLI 工具 - 版本演进规划 (Roadmap)

本项目功能庞大且数据模型复杂，为了确保核心架构稳固、项目能够顺利落地并逐步兑现价值，我们将按照 **V1 (MVP) -> V2 (增强) -> V3 (平台化)** 三个阶段逐步演进。

---

## 阶段一：V1 最小可用版本 (MVP) —— “单兵作战利器”
**目标**：跑通核心数据流，验证标准化双层存储模型（Session/Message）、CLI 交互框架以及全文检索的基础体验。

### 核心功能范围
*   **基础数据架构**：建立基于 SQLite + SQLAlchemy 的 `Session` 和 `Message` 表结构（涵盖 PRD 中的核心字段，其余预留为空）。
*   **CLI 脚手架**：搭建 Typer + Rich 的命令行基础框架（`add`, `import`, `list`, `search`, `show`）。
*   **核心输入源支持**：
    *   优先实现 **OpenAI (ChatGPT)** 官方导出包（`.zip` 或 `conversations.json`）的解析器。
    *   实现通用的 **Markdown 文件** 解析导入。
    *   手工命令快速交互录入单条记录 (`chatasset add`)。
*   **基础存储与检索**：
    *   利用 SQLite **FTS5** 引擎实现对标题、全文的极速关键词检索。
    *   基础的列表查看与按条件（日期、模型）筛选过滤。
*   **基础分类与组织**：实现对会话打标签（单标签、多标签组合）以及标签的增删改查。
*   **基础导出**：将会话内容导出为标准 Markdown，或导出为纯 JSONL 数据格式（为后续模型微调做准备）。
*   **纯本地安全**：完全离线运行，所有数据与缓存只保留在本地磁盘，不进行任何非授权的网络传输。

---

## 阶段二：V2 增强版本 —— “知识提炼工厂”
**目标**：大幅扩展多源接入能力，引入本地大模型辅助总结能力，强化对会话深度加工和二次编辑。

### 核心功能范围
*   **全平台导入适配**：新增 Claude, Gemini, Cursor 以及基于 Shell history 的等常用工具输出格式的解析器。批量扫描并导入多层级目录。
*   **版本化编辑系统**：
    *   允许调用系统编辑器 (如 Vim/Nano) 修改会话与消息体内容。
    *   任何编辑修改都会产生版本快照，支持查看变更历史与回滚（Diff 显示）。
*   **AI 辅助资产提炼**：
    *   接入本地 (Ollama) 或远端 (OpenAI) LLM API。
    *   导入时自动生成一句话摘要、详细总结。
    *   自动提取会话中的 TODO 任务、高频优质提示词 (Prompt)，并标记“最佳回复”。
*   **高级组织与检索**：
    *   引入“项目空间 (Workspace)”或文件夹概念，对会话进行更高维度的隔离与归属划分。
    *   支持特定角色过滤搜索（如：“只搜索 user 发送的内容”）。
    *   基于设定规则的“智能归档路由”。
*   **安全与合规进阶**：
    *   基础的敏感词和隐私数据 (PII，如邮箱、手机号) 正则屏蔽与自动脱敏功能。
*   **统计与看板**：在终端绘制简单的统计图表（聊天量趋势、模型使用占比、高频词云等）。

---

## 阶段三：V3 平台化版本 —— “企业级资产枢纽”
**目标**：引入基于向量的语义级检索，支持更高维度的安全加固，并构建插件生态与高度自动化的数据流。

### 核心功能范围
*   **语义检索引擎**：集成轻量级本地向量数据库（如 ChromaDB 或 LanceDB），实现“语义级搜索”与“相似会话智能推荐”。
*   **复杂对比功能**：深度支持跨会话对比、同一 Prompt 跨模型输出差异对比视图。
*   **安全架构重构**：
    *   实现整库或单会话的高强度加密存储。
    *   引入密码或主密钥 (Master Key) 解锁机制与超时自动锁定。
    *   操作全留痕的审计日志系统与可恢复的“回收站”机制。
*   **知识图谱与链接图谱**：
    *   将离散的对话沉淀为有关联节点的图谱（例如，会话 A 引用了 脚本 B，并且补充了会话 C 的某段 Prompt）。
*   **高级导出与集成**：
    *   自动将筛选后的会话资产库导出打包为完整的静态网站（基于 VitePress 或 MkDocs）。
    *   Webhook 触发导出、邮件采集等复杂自动化流程支持。
*   **开放插件生态**：暴露出标准的 Python Plugin 接口，允许社区开发者提供自定义的外部系统解析器、高级清洗规则与定制化的格式渲染模板。
